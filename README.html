This is a minimal example of a book based on R Markdown and bookdown (https://github.com/rstudio/bookdown), geared specifically for students in ESCI 222 (Sewanee).

 

0.0.1 Setup this bookdown project on your own computer

1. Copy the repo link with the green button above.

2. In Terminal, navigate to your repos folder, then use git clone <paste url> to clone this repo to your local machine.

3. Look inside the bookdown_minimal repo using your Explorer/Finder. Double-click on bookdown-minimal.Rproj to open this “project” in RStudio.

4. If you need to, install the bookdown package.

install.packages("bookdown")

5. In the top right pane (the one that has “Environment” as one of the tabs), look to see if there is now a tab named “Build”.

6. On that “Build”” tab, click on the “Build Book” button, and wait to see if a new window pops up with your book.

7. If so, now look in the bottom-right pane (the one that typically shows plots), under the “Files” tab. Click on the index.Rmd file.

8. In the YAML at the top of index.Rmd, customize the title, author and description of your book. Re-build to see if the changes were applied.

9. If that all worked, then…

(a) Go back to Github and create your own repo for developing your own bookdown project.

(b) Clone it to to your local repos folder.

(c) Now copy and paste all the files from this bookdown-minimal repo into that new empty repo.

(d) Close the RStudio window of the bookdown-minimal project.

(e) In your new repo, change the .Rproj filename from bookdown-minimal to your own preferred filename.

(f) Open up that .Rproj file and start developing your book!

 

0.0.2 How to develop a bookdown book:

In bookdown, the basic rules are:

  1. Each chapter should occur in its own Rmd file;

  2. The top line in each Rmd should be the chapter tile (using a single “#” at the start of the line.

  3. The chapters will be appear in the alphabetical order of their filenames, so include a number at the start of each Rmd filename to control their sequence.

  4. As you work on each Rmd individually, you can render them to polish how they look using the “Knit” button, as you would any Rmd project.

  5. But when you are ready to “render” the entire book into a single product, you should click the “Build Book” button in the top right pane.
